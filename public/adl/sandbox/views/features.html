Delivery
Easy to Deploy
The Sandbox is build completely on HTML5 - there's no need for your users to download anything.
Mobile Friendly
As worldwide adoption of WebGL and HTML5 increases, more and more mobile devices are able to participate in Sandbox worlds. Currently Sandbox works on mobile Chrome, Firefox and Safari, but peformance can be lower.
Simple Networking
Other virtual world platforms have complex networking setup - various ports and firewall rules must be configured to get them working. Sandbox is simple! All you need is access to port 80 - everything is web-based.
hosting
While our demo site is free to use, you can always host your own. It's drop dead simple to install the server
If you're familiar with some basic web server technology, this is all it takes:
> git clone https://github.com/adlnet/Sandbox.git > npm install > node app.js

CDN Support
Since all assets are loaded via a URL that you supply, you can even host 3D files, sounds or textures on a CDN for blazing fast performance.
Terrain
Resolution for Every Need
Our terrain system can render a heightfield down to a single square meter resolution, while still allowing huge spaces up to many square miles.
Easy to Author Formats
You can load a terrain directly from a black and white image file for super-simple authoring, or you can generate a .BT file using common GIS tools.
Texture Splatting
The terrain shader give you 5 channels of detail textures mixed via a user generated mixing map. Each channel can even have its own normal map. We use tri-planer mapping to avoid ugly UV artifacts, and mix the detail maps based on their alpha channels for convincing blending between detail types.

Scripting
JavaScript for Simulation Logic
The language of the web is easy to use, high-performance, and widely known. You can use JavaScript to code all the interaction of your world.
var t = this.transformAPI.getPosition(); if(Vec3.distance(t,[0,0,0]) < .1) alert('You made it!');
Real Time Editing
You can actually see the effects of your scripts right as you work! Just click the play button in the bottom corner, no need to pubish or compile.
Intellegent Support
The Script Editor can give you real time hints about what you can type next, or if your code contains errors.
APIs for Everyone!
We're continously adding new APIs to the scripting system to (safely) expose system functionality to programmers. Nearly everything the system does can be controlled via in-world scripts, but APIs like the TransformAPI or the ClientAPI makes manipulating the system far more transparent.
Physics
Powered by Bullet
Sandbox uses Ammo.js, a port of the popular C++ Bullet library. We use a custom fork of Ammo and Bullet to acheive determisitic playback while bringing high performance into the browser.

Collision Shapes
Spheres, boxes and other primitives all have native support within our physics driver. Imported objects like 3D files can have collision bodies assigned in the editor, and can even use per-polygon collision. You can even build compound collision bodies out of multiple primitives, and use these as high performance proxies for 3D assets.
Joints
Common joint types are also available. You can build complex machines with hinges, sliders, point-to-point constrains and fixed links. Joints can even have limits and motors attached.
PhysicsAPI
All aspects of the scene physics are availble from a special API in the Script Editor. You can even receive notifications about collisions between object, or just move them around with forces.
See Physics Examples
